Module jtag 

Source
Expand description

JTAG replay peripheral model — consumes a recorded remote_bitbang byte stream (the protocol OpenOCD uses to drive an external JTAG bridge) and drives the design’s TCK/TMS/TDI/TRST input pins from it.

Stage 1 of the JTAG cosim plan from discussion #77: deterministic file-replay. The recorded stream is assumed valid and the design’s TDO output is not read back — that lands in stage 2 alongside the TCP-listener mode where TDO has to be routed back over the socket to OpenOCD.

§remote_bitbang byte alphabet

Per OpenOCD’s remote_bitbang.c:

ByteMeaning
'0'..='7'Set (TCK, TMS, TDI) from low 3 bits
'r'TRST off, SRST off
's'TRST off, SRST on
't'TRST on, SRST off
'u'TRST on, SRST on
'R'Read TDO (ignored in stage 1)
'B' / 'b'Blink on / off (no-op)
'Q'Quit (treated as end-of-stream)

§Pacing

OpenOCD has no concept of simulation time. Stream bytes are consumed at chip-clock pace: each byte’s drive values are held on the design’s input pins for hold_edges cosim edges before the next byte is consumed. Default hold_edges = 4 — chosen against the “chip-clock ≥ 2× TCK” assumption that holds by construction in any chipflow design running a debug TAP. Configurable per-cosim via --jtag-hold-cycles <N> for the corner case.
