4//! Jacquard — GPU-accelerated RTL logic simulator.
5//!
6//! Jacquard compiles gate-level netlists into GPU-executable Boolean processor
7//! programs and simulates them on CUDA or Metal GPUs, achieving 5–40× speedup
8//! over CPU-based RTL simulators.
9//!
10//! # Pipeline
11//!
12//! ```text
13//! Verilog netlist
14//!   → NetlistDB        (netlistdb crate — flat gate-level database)
15//!   → AIG              (aig — and-inverter graph with DFFs, SRAMs, clock gates)
16//!   → StagedAIG        (staging — pipeline stages for deep circuits)
17//!   → Partitions       (repcut + pe — hypergraph partitioning → GPU block mapping)
18//!   → FlattenedScript   (flatten — packed GPU instruction stream)
19//!   → GPU kernel        (CUDA or Metal — Boolean processor execution)
20//! ```
21//!
22//! # Key modules
23//!
24//! - [`aigpdk`] — AIGPDK standard cell library interface (AND gates, DFFs, clock gates, SRAMs)
25//! - [`sky130`] / [`sky130_pdk`] — SKY130 cell library support with Liberty timing
26//! - [`gf180mcu`] / [`gf180mcu_pdk`] — GF180MCU cell library support (7t/9t variants; in progress)
27//! - [`pdk`] / [`pdk_decomp`] — PDK-shared types, library detection, and behavioural-model primitives
28//! - [`aig`] — And-inverter graph construction from [`netlistdb::NetlistDB`]
29//! - [`liberty_parser`] — Liberty (.lib) file parser for cell timing data
30//! - [`staging`] — Splits the AIG into pipeline stages via `--level-split` thresholds
31//! - [`repcut`] — Hypergraph partitioning using mt-kahypar
32//! - [`pe`] — Partition executor: maps partitions to GPU block resources (Boomerang stages)
33//! - [`flatten`] — Generates the final packed GPU execution script ([`flatten::FlattenedScriptV1`])
34//! - [`event_buffer`] — Timing event buffer for arrival-time propagation
35//! - [`testbench`] — Testbench configuration and VCD-driven simulation setup
36//! - [`display`] — Display/assertion support infrastructure
